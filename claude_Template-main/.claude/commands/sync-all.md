당신은 `/sync-all` 슬래시 커맨드를 수행 중입니다. 변경 여부와 무관하게 **동기화 체크리스트 전체**를 훑어, 코드 현재 상태와 어긋난 문서/설정을 찾아 맞추는 것이 목표입니다. 릴리스 전이나 큰 변경 후 대청소용입니다.

## 원칙

- 체크리스트의 **단일 소스는 `rules/backend/sync-checklist.md`** 입니다. 그 파일을 읽어 사용합니다.
- 파일을 쓰거나 지우기 전에는 매번 사용자 확인을 받습니다.
- **자동 커밋하지 않습니다.**

## Step 0: 전제 확인

- `rules/backend/sync-checklist.md` 가 없으면 → "동기화 체크리스트가 없습니다 (백엔드/Docker 프로젝트 전용). 종료합니다." 안내 후 종료.

## Step 1: 전수 점검

`rules/backend/sync-checklist.md` 의 표를 읽고, **모든 행**을 파일에 적힌 "Sync order"(code → dependencies → Dockerfile → .dockerignore → docker-compose → .env → README → test docs) 순서대로 점검합니다.

각 대상마다:
- 실제 파일을 읽어 현재 코드 상태와 일치하는지 대조합니다 (추측 금지).
- 어긋난 부분이 있으면 무엇이 다른지 기록합니다.

## Step 2: 보고 + 갱신

점검 결과를 표로 보고:

```
[전체 동기화 점검]
- <대상>: OK / 불일치 (<무엇이>)
```

불일치 항목마다 수정을 제안하고, 사용자 승인 후 갱신합니다 (쓰기 전 확인).
점검 불가 항목(파일 부재 등)은 그대로 표시합니다.
