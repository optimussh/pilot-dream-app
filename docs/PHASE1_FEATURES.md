# Phase 1 Feature List (Balanced Approach)

**목표**: 
- 기존에 있던 기능들을 제대로 Flask 구조로 옮기면서 안정화
- 4대 축(동기부여 / 지식 / 게임 / 현실성)을 **모두 조금씩** 건드려서 앱의 정체성을 보여줌
- 사용자가 "이 앱이 장기적으로 나와 함께 갈 수 있겠구나"라는 느낌을 받을 수 있게 함
- 과도하게 기능을 넣지 않고, **MVP + 확장 가능성**을 동시에 확보

---

## 제안하는 Blueprint 구조 (Phase 1 기준)

아래는 Phase 1에서 현실적으로 필요한 Blueprint 목록입니다.

| Blueprint       | 설명                                      | 우선순위 |
|----------------|-------------------------------------------|----------|
| `main`         | 대시보드, 전체 진행 상황, 환영 화면       | 높음     |
| `radar`        | 관제 레이더 (170대 시뮬레이션)            | 높음     |
| `aircraft`     | 기종설계도 + 카테고리 + 배치도            | 높음     |
| `atc`          | 항공영어 (기존 + 페이지네이션 + 연습)     | 높음     |
| `planner`      | 비행 계획 시뮬레이터                      | 중간     |
| `logbook`      | 가상 조종사 로그북                        | 중간     |
| `gamification` | 뱃지, 비행시간, 기본 챌린지               | 중간     |
| `career`       | 한국 기장 루트 + 비용 + 연봉 정보         | 높음     |

> 참고: `scenarios`, `wisdom` 같은 동기부여 특화 Blueprint는 Phase 2 이후에 분리하는 것을 추천합니다.

## Phase 1 전체 요약 (균형 관점)

- **지식**: 4개 (Radar, Aircraft, ATC, Planner) → 가장 많은 비중
- **게임**: 2개 (Logbook, Badges)
- **동기부여**: 2개 (Dashboard, Daily Wisdom)
- **현실성**: 1개 (Korea Career) → 하지만 중요도가 매우 높음

**총 9개 기능**으로 Phase 1을 구성하는 것을 제안합니다.

이 정도면 "기능이 너무 적다", "너무 많다" 둘 다 피하면서 4대 축의 균형을 어느 정도 맞출 수 있습니다.

## Phase 1 이후 (Phase 2 힌트)

Phase 1이 어느 정도 안정되면 아래 기능들을 고려할 수 있습니다:

- "내가 기장이라면?" 상황극 (Decision Making)
- 미래의 나에게 보내는 편지
- Career Progression Tree (시각적 성장 경로)
- Spaced Repetition 용어 사전
- 주간 챌린지 시스템
